# Contributing

Thanks for helping improve this project — your contributions are welcome.

How to contribute

1. Search existing issues before opening a new one.
2. Use the issue templates for bug reports and feature requests.
3. For code changes, open a pull request with a clear description and tests.

Commit messages

- Use clear, concise messages. Prefer the format: `type: short description` (e.g., `fix: correct typo`).

Local testing

- Follow the repository README for project-specific test/build instructions.

Maintainers

- If you need guidance, open an issue and tag a maintainer.
